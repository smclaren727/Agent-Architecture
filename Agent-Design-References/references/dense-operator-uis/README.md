# Dense Operator UI References

Use this folder for dashboards, consoles, developer tools, and operational products that present a lot of information without becoming noisy.

## What To Look For

- Scan hierarchy.
- Status badge treatment.
- Table/list density.
- Section dividers.
- Inline metadata.
- Error and warning prominence.
- How repeated items differ from page sections.

## Borrow / Avoid Notes

References (2026-07-06): `reference-linear-operator-density`, `reference-vercel-geist-status`.
(Vault reading-room references moved to the `editor-reading/` category.)

Borrow:
- Rows on a flat surface over per-item cards; compact row height; left scan-column + right
  metadata (Linear).
- Value-aware semantic status + monospace paths truncated (not wrapped) (Vercel/Geist).

Avoid:
- Cool/monochrome corporate palettes — keep the warm "Archive" identity.
- Power-user-only density with no labels — keep first-run guidance collapsible.

Implementation caution:
- Vault and Overlay are working tools, not landing pages.
- Prefer dense but organized information over decorative cards.
- Avoid cards inside cards.
