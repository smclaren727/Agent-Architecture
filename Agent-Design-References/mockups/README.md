# Mockups

> **Status (2026-07-06): Phase 2 — deferred.** The Phase-1 *architecture* direction was chosen
> directly from the current apps and real references (see `selected-direction/direction-brief.md`),
> not from generated mockups. This folder is for **Phase 2**: exploring the distinctive *visual*
> direction (glass depth, motion character, accent posture) layered on top of the locked Phase-1
> architecture. Treat it as optional visual exploration, not a prerequisite.

Use this folder for generated direction mockups and iteration notes.

## Direction Set

Generate a small set of polished directions before choosing one:

1. Quiet macOS glass.
2. Dense operator console.
3. Warm knowledge workspace.
4. Minimal command center.
5. High-contrast dark productivity.

Each direction should include real app surfaces:

- Vault with notes plus Chat.
- Vault Conventions view.
- Overlay Automations with Runner status.
- Menus or popovers.
- Empty/error/loading state treatment.

## Evaluation

For each mockup, write:

- What feels right.
- What feels wrong.
- Which details should become tokens.
- Which details are decorative and should be ignored.
- Whether it would still work at real app density.
