# Imagegen Prompt Starting Points

> **Phase 2 (deferred).** Phase-1 locked the *architecture* (`selected-direction/direction-brief.md`)
> without mockups. Use these prompts only for Phase-2 *visual* exploration on top of that
> architecture — and constrain them to it: keep the "Archive" identity, the two-speed density
> (Vault reading-room / Overlay operator), semantic status, glass-for-overlays, and native-feel
> shell. Any generated direction that fights the Phase-1 architecture is out of scope.

These prompts are starting points for generated UI direction mockups. Use references from `../references/` as style inputs when available.

## Shared Constraints

Use these constraints in every prompt:

```text
Use case: ui-mockup
Asset type: high-fidelity product UI direction mockup
Primary request: create a polished visual direction for the Agent-Vault and Agent-Overlay desktop apps.
Subject: real productivity app screens with dense but readable controls, side panels, right docks, menus, status badges, editors, and chat surfaces.
Must preserve: Markdown/YAML/plain-file source-of-truth feeling, operator clarity, readable text hierarchy, accessible contrast, compact controls, real app density.
Avoid: marketing landing page layout, oversized hero sections, decorative cards inside cards, unreadable blur, pure purple/blue gradient theme, generic SaaS dashboard stock look.
```

## Direction 1: Quiet macOS Glass

```text
Create a high-fidelity desktop app mockup for Agent-Vault and Agent-Overlay in a quiet macOS glass style.
Show Vault with a notes sidebar, Markdown editor, and right Chat dock with runtime and permission controls.
Show Overlay with an Automations console: trigger list, Runner liveness, recent runs, and compact service controls.
Use translucent panels with restrained blur, thin borders, strong text contrast, subtle shadows, compact menus, and muted accent colors.
The UI should feel native, calm, inspectable, and professional.
```

## Direction 2: Dense Operator Console

```text
Create a high-fidelity desktop app mockup for a dense operator console visual direction.
Show Overlay Automations as the primary surface with trigger rows, status badges, exact recent runs, service controls, and validation messages.
Include Vault as a sibling knowledge workspace surface with notes, Conventions findings, and Chat.
Use crisp separators, compact row density, restrained color, clear hierarchy, and minimal decoration.
The UI should feel fast to scan during repeated work.
```

## Direction 3: Warm Knowledge Workspace

```text
Create a high-fidelity desktop app mockup for a warm knowledge workspace visual direction.
Show Vault as the primary surface: notes, graph/context details, Conventions findings, and Chat suggestions.
Include Overlay as a smaller operator panel for automations and runs.
Use balanced warm-neutral surfaces, clear readable typography, soft but not decorative elevation, and subtle accent color.
The UI should feel calm, thoughtful, and still technical.
```

## Direction 4: Minimal Command Center

```text
Create a high-fidelity desktop app mockup for a minimal command-center direction.
Show Vault and Overlay with compact top controls, command-like menus, narrow side panels, and clear mode selectors.
Use low visual noise, strong alignment, tight spacing, simple status indicators, and focused overlays.
The UI should feel precise and efficient.
```

## Direction 5: High-Contrast Dark Productivity

```text
Create a high-fidelity desktop app mockup for a dark productivity interface with high contrast and subtle depth.
Show Vault Chat with a long prefilled prompt, permission controls, and transcript actions.
Show Overlay Automations with liveness status, trigger lifecycle controls, and recent runs.
Use dark layered surfaces, semantic status colors, crisp borders, accessible contrast, and restrained glow only for focus or active state.
The UI should feel polished without becoming cyberpunk or decorative.
```
